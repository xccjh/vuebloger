  • async和await的错误捕获

    • 案发现场
      • 尝试捕获它
        • try
        • catch

    # 案发现场

    举一个很常见的例子：

    function getData(data) {
  return new Promise((resolve, reject) => {
    if (data === 1) {
      setTimeout(() => {
        resolve('getdata success')  // 👈
      }, 1000)
    } else {
      setTimeout(() => {
        reject('getdata error')  // 👈
      }, 1000)
    }
  })
}
window.onload = async () => {
  let res = await getData(1)
  console.log(res) //getdata success
}

    这样写可以正常打印getdata success 但是如果我们给getData传入的参数不是1，getData会返回一个reject的Promise，而这个地方我们并没有对这个错误进行捕获，三秒后则会在控制台看见这样一个鲜红的报错Uncaught (in promise) getdata error

    # 尝试捕获它

    # try

    window.onload = async () => {
  try {
    let res = await getData(3)
    console.log(res)
  } catch (error) {
    console.log(error) //getdata error
  }
}

    如果抛出错误

    function getData(data) {
  return new Promise((resolve, reject) => {
    if (data === 1) {
      throw new Error('error')  // 👈
      setTimeout(() => {
        resolve('getdata success')
      }, 1000)
    } else {
      setTimeout(() => {
        reject('getdata error')
      }, 1000)
    }
  })
}

window.onload = async () => {
  try {
    let res = await getData(1)
    console.log(res)
  } catch (error) {
    console.log(error) // Error: error
  }
}

    或许我们可以用一个trycatch将所有的await包起来，但是这样就很不方便对每一个错误进行对应的处理，还得想办法区分每一个错误。

    # catch

    因为返回的是一个Promise，那我们首先想到的就是.then()和.catch()，于是很快就能写出以下代码：

    window.onload = async () => {
  let res = await getData(3).then(r=>r).catch(err=>err);
  console.log(res) //getdata error
}
window.onload = async () => {
  let res = await getData(1).then(r=>r).catch(err=>err);
  console.log(res) // Error: error
}
